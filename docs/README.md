# Cubby Documentation

Keep the repository root limited to project entry points such as `README.md` and `AGENTS.md`. Feature plans, technical designs, implementation notes, and historical reports belong under `docs/`.

## New Feature Plans

- Put plans in `docs/plans/`.
- Name each file after the feature, for example `docs/plans/import-export.md`.
- Avoid generic names such as `plan.md`, `spec.md`, `prd.md`, or `todo.md` at the repository root.
- Link the relevant GitHub issue near the top when one exists.
- Update the plan as decisions change so checked-in guidance remains useful.

Existing documents outside `docs/plans/` are retained as historical design and implementation context. Move them only when actively revising that documentation.
